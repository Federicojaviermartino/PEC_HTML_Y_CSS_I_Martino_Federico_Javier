M2. Un workflow moderno de desarrollo front-end
En este segundo módulo de la asignatura Herramientas HTML y CSS I empezaremos a escribir código y nos familiarizaremos con algunas de las herramientas que son imprescindibles hoy en día para hacer desarrollo web de forma eficiente.

Si alguna vez habéis desarrollado la interfaz de una página web, habréis trabajado con un editor de texto editando ficheros HTML, CSS y JavaScript. Actualmente, los desarrolladores web utilizamos, además de un editor de código, una serie de herramientas algo más complejas. Si bien al principio puede parecer que complica un proceso que es relativamente sencillo, el uso de herramientas avanzadas facilita a la larga el desarrollo, ya que permite la gestión de dependencias, la automatización de ciertas tareas repetitivas y el uso de lenguajes y utilidades más avanzadas, entre otras cosas.

Para poder usar estas herramientas, será necesario que os familiaricéis con el terminal. Aprenderemos cómo navegar por la estructura de directorios y ejecutar órdenes desde el terminal.

A continuación nos plantearemos qué flujo de trabajo (más conocido como workflow) necesita un proyecto en función de sus características. Una vez hayamos definido ese flujo básico, aprenderemos a usar algunas de las herramientas que nos permitirán construirlo. Por ejemplo, para la gestión de dependencias emplearemos npm, para compilación de código emplearemos Parcel, etc. y veremos para qué sirve cada una.

Finalmente, desarrollaremos el workflow diseñado, y crearemos una plantilla de estructura de ficheros y un conjunto de herramientas con Parcel, que nos servirá para desarrollar futuras aplicaciones. Se convertirá, pues, en el punto de inicio desde donde se realizarán las PECs de este curso, que partirán del código que generemos en este módulo.

Como el producto de esta asignatura será una página web, durante este módulo también la publicaremos en internet.

Como siempre, los materiales de aprendizaje, disponibles en la sección Recursos básicos, y en la sección Contenidos y actividades encontraréis diferentes ideas y propuestas de actividades sencillas para consolidar los conocimientos adquiridos con los recursos propuestos. Si tenéis cualquier duda sobre alguna de las actividades propuestas o no sabéis la respuesta a alguna de las preguntas, abrid un hilo en el foro y lo comentamos entre todos.

Objetivos de aprendizaje
Familiarizarse con el uso del terminal.
Conocer el funcionamiento y la utilidad de los gestores de paquetes como npm.
Conocer el funcionamiento y la utilidad de los module bundlers como Parcel.
Conocer las diferencias entre entornos de desarrollo y de producción.
Crear una plantilla de inicio con Parcel que permita desarrollar proyectos web basados en un workflow moderno.
Conocer los pre- y postprocesadores de código.
Publicar una web en internet.
