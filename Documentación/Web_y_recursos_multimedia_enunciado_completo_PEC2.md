P2. Web y recursos multimedia (enunciado completo)
Actividad evaluable

En los primeros módulos, y en concreto en la primera entrega, hemos creado un entorno moderno para desarrollar aplicaciones web, e incluso lo hemos utilizado para crear un sitio web.

Así pues, esta práctica propone el reto de seguir añadiendo funcionalidades sobre el código desarrollado en la primera PEC y el conocimiento adquirido en el Módulo 3. La práctica tiene unos requisitos mínimos que se especifican en el apartado de Descripción. Todo el proceso que sigas deberá estar documentado, y habrá que entregar exactamente lo que se pide en el apartado de Formato antes fecha de la fecha límite entrega para poder ser evaluado.

Recuerda que el peso de esta PEC es de un 35% de la nota de la evaluación continua.

Objetivos
Añadir adecuadamente recursos multimedia en nuestro sitio web responsive.
Utilizar el inspector DOM, el editor de estilos y las herramientas para diseño responsive del navegador como ayuda para el desarrollo.
Implementar adecuadamente las técnicas de clipping, imágenes responsive y animación web, y hacerlo de forma accesible.
Descripción
La PAC consta de dos partes: desarrollo del sitio web y documentación. Es necesario realizar y entregar ambas partes de la PAC para que sea evaluada.

 

Parte 1: Desarrollo

En la parte de desarrollo, deberás seguir trabajando en el proyecto web de la PAC anterior, siguiendo las directrices marcadas. No obstante, puedes hacer las modificaciones que consideres oportunas mientras mantengas la misma estructura y los requisitos definidos en la PAC 1.
Utiliza el mismo repositorio de GitHub y publica la web bajo el mismo dominio.

El sitio web deberá cumplir las características que se especifican a continuación:

Todas las imágenes de la web deben ser responsive y al menos debe haber una imagen por cada técnica de imagenes responsive.
Utiliza sharp para optimizar todas las imágenes de la web. Considera, si lo crees necesario, cambiar los formatos utilizados como parte del proceso de optimización.
Haz una tabla con la mejora de cada una de las imágenes optimizadas especificando el nombre de la imagen, el formato antiguo, el formato nuevo y la mejora obtenida en el peso. Analiza los resultados. Añade esta tabla y el análisis en la sección 1 de la documentación.
La web debe ser responsive y visualizarse correctamente desde cualquier dispositivo moderno (teléfono, tableta, ordenador, etc.). Tal como hemos visto en el Módulo 3, esto implica que los contenidos multimedia (especialmente imágenes) deben adaptarse al dispositivo que consulta la web: es importante que se ajusten, como mínimo, al tamaño de la pantalla. De nuevo, y sabiendo que normalmente queremos un diseño lo más limpio posible, intenta desarrollar una página que permita mostrar lo aprendido.
La web debe tener algunos elementos animados mediante transiciones y animaciones aplicadas en CSS: pueden ser imágenes, enlaces, textos u otros recursos. ¡Sed creativos!
Las prácticas entregadas deben cumplir con las pautas AA de WCAG 2.0 (http://www.sidar.org/traducciones/wcag20/es/#guidelineLinks to an external site.Links to an external site.). En particular, dentro de los objetivos de esta actividad, esto incluye (pero no se limita a):
ofrecer alternativas al contenido no textual,
no usar únicamente el color para comunicar información,
asegurar el contraste mínimo de color,
proporcionar enlaces “saltar navegación” y equivalentes donde haya bloques de contenido repetidos entre páginas,
utilizar correctamente el título de página, encabezados, y dar textos adecuados a los enlaces siguiendo las buenas prácticas especificadas en los recursos de aprendizaje,
especificar correctamente el idioma del contenido de cada página y de los fragmentos en otros idiomas que pueda haber,
si hay formularios y/o tablas, deben ajustarse a las buenas prácticas especificadas en los recursos de aprendizaje,
las páginas deben ser válidas.
Debéis leer todas las pautas AA y asegurar de que vuestra entrega las cumple.

 

Portada

En esta página debe haber al menos un recurso gráfico hecho con SVG. Este SVG debe tener una pequeña animación mediante CSS.
Debe haber al menos un recurso gráfico editado con clip-path. ¡Sé creativo! Recorta alguna imagen y crea alguna forma relativamente compleja que quede bien integrada en la página.

Parte 2: Documentación
Documenta el proceso de desarrollo que has seguido para crear tu proyecto. Justifica las decisiones que has tomado y analiza los resultados.

Genera la documentación siguiendo la estructura propuesta a continuación:

Formatos de imagen utilizados
Tabla de imágenes optimizadas. Especifica el nombre de la imagen, el formato antiguo, el formato nuevo y la mejora obtenida en el peso. Analiza los resultados.
Utilización de las técnicas de imagen responsive explicadas en el módulo 3
Animación de elementos en CSS
Uso de clip-path
Semántica y accesibilidad
La documentación es un documento que ayuda a dejar constancia de los desarrollos realizados (tanto para futuros colaboradores del proyecto como para nosotros mismos cuando tengamos que actualizarlo). Cuida el formato y la legibilidad.

Recomendaciones para la documentación:
• Sé claro y conciso en las explicaciones.
• No es necesario documentar impresiones personales ni subjetivas.
• Justifica lo que expliques y hazlo con rigor.
• Si utilizas capturas de pantalla, asegúrate de que se entienda qué quieres mostrar (no pongas una imagen con 80 líneas de código).
• Si muestras código o comandos, utiliza estilos para destacarlos del resto del texto.
• Procura que la documentación sea legible, con una tipografía clara y respetando interlineados y márgenes.
• Añade títulos para cada sección y jerarquiza el contenido utilizando los estilos adecuados (tamaño de fuente, negrita, subrayado, etc.).
• Si consideras necesario añadir muchas imágenes, valora la posibilidad de incluirlas en un anexo.
• No superes las 12 páginas.

Puedes encontrar más información sobre cómo documentar en: http://www.lcc.uma.es/~eat/pdf/sw-spanish.pdfLinks to an external site.
